                  Install Cine Power Planner

                  Add the planner to your Home Screen to use it like an app.

                  1. Open Cine Power Planner in your browser.
                  2. Open the browser menu.
                  3. Choose Add to Home Screen and confirm.

                  After installation, launch Cine Power Planner from your Home Screen.

                  Move your existing data into the app

                  Safari keeps the browser tab and the installed app separate. To move your Cine Power Planner data into the app:

                  1. Open Cine Power Planner in Safari where your projects are still visible.
                  2. Go to Settings → Backup & Restore → Backup to download a copy of your data.
                  3. Open Cine Power Planner from your home screen.
                  4. Go to Settings → Backup & Restore → Restore and pick the backup file.

                  After restoring once, all changes stay inside the installed app. Keep the backup if you plan to reinstall or switch devices.

                    Quick start checklist

                    1. Name your project in the Project Name field and press Save (or press Enter/Ctrl+S) to capture the current rig before experimenting—saved entries stay available offline.
                    2. Walk through Configure Devices , the Power Summary and the Connection Diagram in order. Each section updates immediately so you can verify power draw, cable routing and runtime checks at a glance.
                    3. Before wrapping up, download an Export project JSON and a full-app Backup so you have two offline recovery copies that can be restored from the Backup & Restore panel.
                    4. Confirm the offline indicator glows before disconnecting and, if an update is pending, trigger a Force reload while you are still online so the latest locally stored Uicons, fonts and help articles finish caching.
                    5. Once your exports are saved to two destinations, open Restore rehearsal to load the bundle in a sandbox, verify every project, rule and device matches the live profile, then close the rehearsal so the production data stays untouched.

                    Tip: Keep backups on separate drives when travelling— Restore always creates a fresh safety copy so nothing is lost when you undo a change.

                    Protect your work

                    • Press Save after meaningful tweaks—manual saves are instant, work offline and make it easy to roll back using the project switcher.
                    • Enable Show auto backups in project list to surface the background snapshots captured every ten minutes and after device imports.
                    • Download redundant copies whenever you finish a milestone: Export Project for the active rig, Backup for global preferences and auto gear, and the Device Library Export for custom gear.
                    • Store exports on at least two locations (for example SSD + cloud sync) before updating browsers, clearing caches or installing on another computer, then verify restores from the Restore panel while you still have the originals.
                    • Before approving an update toast or running Force reload, capture a fresh manual backup and store the download alongside your project exports so no data is lost if a cache clears mid-update.

                    Manual backups bundle a timestamped copy of every saved project, favorite, rule and preference. Restores make a safety snapshot first so you can undo any mistake without losing data.

                    Restore rehearsal drill
                    1. Open Settings → Restore rehearsal and choose whether you're testing a full-app backup or a shared project bundle.
                    2. Use Select file to load the JSON export from your offline media. The sandbox keeps the live profile read-only while it inspects the contents.
                    3. Review the comparison table and automatic gear diff for discrepancies. Investigate anything unexpected before promoting the backup.
                    4. When the rehearsal looks correct, exit with Abort rehearsal or, if you're ready to apply it, proceed with Continue rehearsal restore knowing the system already captured a safety snapshot you can roll back to.
                    Version comparison drill
                    1. Open Settings → Compare versions to launch the diff workspace.
                    2. Select a baseline manual save and the newest auto backup, then review the summary and detailed diff for unexpected changes before proceeding.
                    3. Capture context in Incident notes and export the log with Export log . Store the JSON alongside your backups.

                    Project data safety checklist

                    1. Save the active setup under a new name before experimenting so earlier versions stay available for comparison.
                    2. Download both a full-app Backup and the current project JSON (via Export project ) immediately after important updates so you have self-contained copies you can move offline. Annotate filenames with the project, operator, location and timestamp to preserve context.
                    3. Store at least two copies of every file (for example, on your laptop and an external SSD) to guard against device loss when you are travelling or on set. Note where the redundant copies live so handoffs remain traceable.
                    4. Open each project JSON through Import project inside a private or secondary profile that stays offline to confirm it loads correctly; close the dialog or skip saving if you are only testing the archive.
                    5. Periodically test full-app backups by loading them with Restore rehearsal first. Check the comparison table and automatic rule preview, stage clean rehearsals with Continue rehearsal restore and cancel suspicious ones with Abort rehearsal . Only run Restore once the sandbox checks out—the planner still creates a fresh safety snapshot beforehand so you can revert instantly if needed. Log which backup you restored and where the validation copy lives.
                    6. Capture the runtime guard during every rehearsal. Open the developer console, confirm window.__cineRuntimeIntegrity.ok is true (or rerun window.cineRuntime.verifyCriticalFlows({ warnOnFailure: true })) and store the output with your verification notes so the audit trail proves every save/share/restore gateway stayed online-free.

                    Tip: Rename archived files with the project name and date (for example, “2024-06-15 camera-rig backup.json”) and mark trusted copies as read-only so the newest verified export stays obvious when you need to recover or share offline.

                    Console verification commands

                    • Open the developer console and inspect window.__cineRuntimeIntegrity. The guard should report { ok: true } with an empty missing list before you rely on local saves, shares or restores.
                    • Run window.cineRuntime.verifyCriticalFlows({ warnOnFailure: true }) whenever you finish a rehearsal or incident review. The refreshed report confirms every persistence, offline and UI safeguard stayed available without connectivity.
                    • Use window.cinePersistence.storage.exportAllData() to preview the exact JSON captured by planner backups. Compare project counts, timestamps and file size before archiving exports to redundant media.
                    • Validate incoming bundles with window.cinePersistence.share.decodeSharedSetup(payload) before you import them on production data. Pair it with window.cinePersistence.share.applySharedSetup() inside a rehearsal profile to rehearse restores without touching live saves.

                    Tip: Capture the console output alongside your backup filenames so audits prove which commands you checked before travelling or rotating media.

                    Gear List

                    • The Generate Gear List and Project Requirements button expands every selected device and project detail into a rich table grouped by category and quantity.
                    • Any change to devices, lenses, batteries or project information rebuilds the list so it always reflects the current configuration.
                    • The generator assembles the table step by step:
                      1. Each chosen device is cloned so accessories and rules can be applied without altering the source data.
                      2. Default accessories such as power cables or mounting brackets are attached according to each device definition.
                      3. Scenario rules from Required Scenarios inject rigging and weather protection.
                      4. Entries from the Project Requirements form contribute additional hardware and notes.
                      5. An internal slug (manufacturer + model) is generated; matching slugs are merged and their quantities summed.
                      6. Lines are sorted into Camera, Lens, Power, Monitoring and Accessories sections and then alphabetically.
                      7. Tooltips list weight, dimensions and other metadata for quick reference.
                    • Accessory packs ensure common dependencies are covered:
                      • Handheld + Easyrig adds a telescopic handle, shoulder rig and compatible quick‑release plates.
                      • Gimbal supplies the selected gimbal, balancing accessories, power feed cables and any required filters or sunshades.
                      • Outdoor adds spigots, umbrellas, CapIt rain covers and water‑resistant cable wraps for monitor protection.
                      • Vehicle or Steadicam scenarios append suction mounts, hard‑mount plates or isolation arms where defined.
                    • Lens selections append front diameter, weight and rod requirements, introduce lens supports and matte box hardware, and warn about incompatible rod standards or filter trays.
                    • Battery entries mirror counts from the power calculator; if draw exceeds safe limits the list adds hotswap plates or the selected hotswap device.
                    • Monitoring preferences contribute default screens for each role (Director, DoP, Focus Puller, Client), matching video transmitters and pre‑bundled cable sets.
                    • The Project Requirements form acts as a template for the list. Each field adds context and gear:
                      • Project Name, Production Company, Rental House, DoP, Prep and Shooting Days populate the printed header and can trigger weather gear with outdoor scenarios.
                      • Crew entries keep names, roles and email addresses attached so contact lists and call sheets stay current.
                      • Delivery Resolution, Recording Resolution, Sensor Mode, Aspect Ratio, Codec and Base Frame Rate lock in capture specs so printed summaries and exports show the intended format.
                      • Required Scenarios toggle accessory packs as described above.
                      • Lenses define the core set for the shoot, surface front diameters for matte-box adapters and inform balancing accessories when scenarios like gimbals are active.
                      • Camera Handle, Viewfinder Extension and User Button Layouts supply matching handles, VF extensions and quick-reference notes for camera, monitor and viewfinder controls.
                      • Matte Box and Filter Set selections insert filter trays, donuts, adapters and storage cases.
                      • Monitoring Configuration, Frame Guides, Aspect Mask Opacity and Video Distribution choices attach wireless links, overlays, splitters, antennas and channel‑assignment notes.
                      • Viewfinder Settings and Tripod Preferences add EVF mounts, extension brackets, tripod heads, spreaders and counterbalance details.
                    • Toggle Highlight automatic gear to temporarily tint rows injected by rules. Badges on each affected line list the rule names so you can audit automations at a glance.
                    • The resulting gear list appears in printable overviews and exported project files so collaborators see full context.
                    • Use the Delete Gear List control (also available inside printable overviews) when you need to rebuild from scratch—the planner double-checks, saves an automatic backup and then clears the saved gear table and project requirements safely.
                    • Gear lists save automatically with the project; Export downloads a JSON file, Import restores it and Delete removes the saved list and shows the generator again.
                    • Gear list forms use fork buttons to duplicate your entries instantly, making alternate configurations easy to explore.
                    • Review the Project Requirements summary alongside the Gear List output to confirm every selection is represented.

                    Automatic Gear Rules

Every chosen condition must match the project before the automation runs, and at least one condition is required to save a rule.
                    • Rules execute after the planner assembles the gear list, so your adjustments stack on top of built-in accessories and scenario packs.
                    • Switch on Highlight automatic gear while reviewing the gear list to wash rule-driven rows with a temporary color overlay. The badges that appear beside each tinted line list the rule names that fired so you can trace changes instantly.
                    • Click Add rule to reveal the editor, name the rule for quick reference and choose the conditions that must be active before it triggers.
                    • Use Add these items to insert gear lines with category and quantity; pick Custom Additions for notes, bespoke equipment or reminders.
                    • The rules list keeps every automation visible with duplicate, reorder and delete controls so you can maintain complex setups quickly.
                    • The Rule coverage overview dashboard surfaces duplicate triggers, potential conflicts, uncovered scenarios and net add/remove counts. Its buttons filter the rules list for offline audits and every export carries the same annotations for printed or shared reports.
                    • Remove these items hides entries the generator would normally include—match the item name and category shown in the gear list to target the right row.
                    • Tap Reset to factory additions whenever you want to restore the default helper rules that ship with the planner.
                    • Save frequently used combinations as presets via the Preset menu, then switch or delete them as your workflow evolves.
                    • Background automatic backups capture rule changes every 10 minutes. Toggle Show automatic backups to reveal the timeline inside Automatic Gear Rules; when you hide it again the status notice under the toggle confirms snapshots are still running in the background and how many entries remain so you can bring them back before restoring.
                    • Tune the Backup retention field to decide how many automatic snapshots stay available on this device. The live summary warns before trimming older versions so you can export rule sets you still need.
                    • When backups are visible, pick a timestamped snapshot from the Choose a backup dropdown to review its details and press Restore to roll back to that version. Each entry lists how many rules it contains so you can recover confidently even while offline.
                    • Export rules downloads a JSON copy of your automation set; Import rules loads an existing file so crews can sync configurations without relying on connectivity.
                    • Rules travel with backups and exports. Include them when sharing a project, then decide on import whether to apply the shared rules only to the opened project or add them to all projects.

                    • Warnings appear if the configuration exceeds battery output limits—watch the main output, D-Tap and hotswap alerts for guidance before committing to a battery plan.
                    • Select a Battery Hotswap to factor in dual plates or adapters; its capacity and pin limits are merged into the calculation.
                    • The optional Battery Comparison panel shows runtimes for all batteries at once so you can compare options side by side.

                    Calculation Details

                    • Each device contributes its power draw in watts to Total Consumption, which you can inspect in the Power Summary panel.
                    • Current at 14.4 V (33.6 V for B‑Mount) and 12 V (21.6 V for B‑Mount) equals total watts divided by voltage.
                    • Runtime = battery capacity (Wh) ÷ total consumption.
                    • User-submitted runtimes collected through Submit User Runtime Feedback are adjusted for temperature and weighted before averaging:
                      • Temperature: ×2 at ≤−20 °C, ×1.6 at ≤−10 °C, ×1.25 at ≤0 °C.
                      • Resolution: ≥12K ×3, ≥8K ×2, ≥4K ×1.5, ≥1080p ×1; lower scales with 1080p.
                      • Frame rate scales linearly from 24 fps (e.g. 48 fps = ×2).
                      • Wi‑Fi adds 10 %; monitor entries below spec are weighted by their brightness ratio.
                      • Codec factors: RAW/BRAW/ARRIRAW/R3D/CinemaDNG/Canon RAW/X‑OCN ×1; ProRes ×1.1; DNx/AVID ×1.2; All‑Intra ×1.3; H.264/AVC ×1.5; H.265/HEVC ×1.7.
                      • The final weight reflects each device's share of total power draw.
                    • Warnings highlight when current approaches or exceeds main or D‑Tap output ratings—check the main output warning and D-Tap warning rows for specific advice.

                    ?FAQ

                    How do I save a project?

                    Cine Power Planner autosaves every change to protected local storage. The status dot beside the Project Name field glows while that happens so you always know the latest update was captured. For a named snapshot:

                    1. Enter a descriptive name and click Save or press Enter / Ctrl+S (⌘+S on macOS).
                    2. Confirm it appears in the Saved Projects list. Loading another entry brings it back instantly while leaving the autosaved draft untouched.
                    3. Repeat the save action any time to update the snapshot, or choose -- New Project -- when you want to start fresh without deleting earlier saves.

                    Saved entries stay offline on your device, sync into manual exports and can always be restored, so no project data is lost between sessions.

                    How do I share or back up a project?

                    Use the exporter to create portable archives that protect every configuration detail:

                    1. Click Export Project and choose a secure file name. Include automatic gear rules when collaborators should inherit your automation presets.
                    2. Store the downloaded JSON bundle wherever you keep backups or send it to teammates. It contains the project requirements, gear list, runtime analysis, linked auto-backups and any custom devices needed to rebuild the setup.
                    3. To restore, paste or select the file inside the Import Project field and press Import. The planner validates the bundle and asks how to apply gear rules so nothing overwrites your library without approval.

                    Keep regular exports as part of your backup rotation; imports never delete existing saves, so you can maintain as many restore points as you need.

                    How do automatic backups work?

                    Cine Power Planner layers three automatic safety nets alongside your manual saves:

                    • Auto snapshots capture the active project every 10 minutes. Enable Show auto backups in project list in Settings → Backup & Restore to reveal the timestamped auto-backup-… entries beneath the Saved Projects dropdown so you can load, review and rename them.
                    • Hourly full-app backups download JSON archives (for example, “2024-06-15T14-00-00 full app backup.json”) that bundle every project, automatic gear rule, device edit, favorite and runtime log. Keep at least two copies on separate drives so you always have an offline recovery point.
                    • Restore safeguards run whenever you restore or perform a factory reset. The planner creates a fresh “before” backup automatically so you can revert immediately if the new file is not what you expected.

                    Open any auto snapshot to inspect it; press Save if you want to keep the snapshot as a regular project. Archive the hourly downloads once verified, or delete them after you have stored redundant copies elsewhere.

                    How do Automatic Gear Rules work?

                    Open Settings → Automatic Gear Rules and click Add rule to reveal the editor. Name the rule so it is easy to recognise later and pick the conditions that must be active before it runs.

                    Conditions can watch Required Scenarios, Mattebox options, Camera handles, Viewfinder extension, Video distribution, Camera selection, Onboard monitors, Wireless transmitters, FIZ motors, FIZ controllers and FIZ distance devices. The rule runs only when every chosen value is active in the Project Requirements form or selected in the Configure Devices panel. Add gear lines with category and quantity in Add these items—choose Custom Additions for notes or bespoke kit.

                    Use the Remove these items list to hide entries the generator adds by default. Rules are stored in Settings, included in backups and apply to printable gear lists and exported project bundles. Use Reset to factory additions to restore the built-in helpers, or download and reload JSON sets with Export rules and Import rules when collaborating offline.

                    Factory additions include an every five shooting days consumables kit that adds four Bluestar eye leathers, two rolls of Pro Gaff tape per color slot, four clapper sticks, two Kimtech wipe sleeves and a Sprigs Red 1/4" pack for each interval. Longer shoots automatically scale those quantities so you can reset to the defaults and keep continuity gear stocked without editing rules by hand.

                    How do I edit device data?

                    Click the Edit Device Data… button to open the database editor where you can add, change or remove devices.

                    How is battery runtime calculated?

                    The app divides the selected battery's capacity (in watt-hours) by the total power consumption of your devices. For example, a 98 Wh battery powering a 24 W project is estimated to run for about 4 hours.

                    What do the warning colors mean?

                    Orange messages indicate the project is nearing the battery's output limit. Red warnings mean the estimated current exceeds the safe rating and could damage the battery.

                    Where are my projects saved?

                    Projects, project requirements, automatic gear rules, preferences and runtime feedback live in your browser's localStorage so they remain available offline. The active project autosaves after every change and the planner records background safety snapshots you can reveal via Show auto backups in project list . Download the active setup with Export Project, capture a full planner snapshot through Backup and use the Device Database Editor's Export / Import controls for custom gear. Restores always create a safety copy first so you never lose data while testing archives.

                    Does the planner work offline?

                    Yes. Once the fonts and database are cached on first load, all features continue to work without an internet connection, including saving, autosaving, backups, restores and device imports. The Offline indicator confirms when the app is running locally, and you can refresh assets without losing saved data through Force reload whenever you reconnect. Keep your latest exports handy so every workflow—on set, in transit or in the studio—has an immediate offline recovery path.

                    How do I move everything to another computer?

                    Prepare both devices before you leave the original workstation:

                    1. Save each project and download the active setup with Export Project.
                    2. Open Settings → Backup & Restore and download a full-app Backup so preferences, automatic gear rules, custom devices and runtime feedback travel with you.
                    3. Export custom gear from the Device Database Editor using the Export button.
                    4. Copy the JSON files to redundant storage (two drives or a drive plus cloud sync) before leaving.
                    5. On the destination machine, open the planner once while connected (or complete the install guide ) so assets cache for offline use.
                    6. Restore the planner snapshot through Restore , then import project JSON files or Device Library exports as needed.

                    Keep the original backup untouched until you confirm the projects and device entries load correctly on the new system.
