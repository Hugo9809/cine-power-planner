                  Install Cine Power Planner

                  Add the planner to your Home Screen to use it like an app.

                  1. Open Cine Power Planner in your browser.
                  2. Open the browser menu.
                  3. Choose Add to Home Screen and confirm.

                  After installation, launch Cine Power Planner from your Home Screen.

                  Move your existing data into the app

                  Safari keeps the browser tab and the installed app separate. To move your Cine Power Planner data into the app:

                  1. Open Cine Power Planner in Safari where your projects are still visible.
                  2. Go to Settings → Backup & Restore → Backup to download a copy of your data.
                  3. Open Cine Power Planner from your home screen.
                  4. Go to Settings → Backup & Restore → Restore and pick the backup file.

                  After restoring once, all changes stay inside the installed app. Keep the backup if you plan to reinstall or switch devices.

                    Protect your work

                    • Press Save after meaningful tweaks—manual saves are instant, work offline and make it easy to roll back using the project switcher.

                    • Store exports on at least two locations (for example SSD + cloud sync) before updating browsers, clearing caches or installing on another computer, then verify restores from the Restore panel while you still have the originals.
                    • Before approving an update toast or running Force reload, capture a fresh manual backup and store the download alongside your project exports so no data is lost if a cache clears mid-update.

                    Restore rehearsal drill
                    1. Open Settings → Restore rehearsal and choose whether you're testing a full-app backup or a shared project bundle.

                    3. Review the comparison table and automatic gear diff for discrepancies. Investigate anything unexpected before promoting the backup.
                    4. When the rehearsal looks correct, exit with Abort rehearsal or, if you're ready to apply it, proceed with Resume restore rehearsal knowing the system already captured a safety snapshot you can roll back to.

                    Record the filename, timestamp and verification results in your backup log before closing the rehearsal so every drill leaves behind a documented recovery point.

                    Version comparison drill

                    Automatic safety copies

                    • Auto snapshots capture the active project every 10 minutes or after roughly 50 edits, and rapid adjustments are batched so bursts closer than a second only advance the counter once. When you switch projects, import or export or prepare to reload, the planner captures a fresh snapshot even if that cadence hasn't elapsed. Enable Show auto backups in project list to reveal the timestamped auto-backup-… entries beneath your regular saves and convert any snapshot into a permanent project with Save.
                    • Hourly full-app backups download JSON archives named with the current time (for example, “2024-06-15T14-00-00 full app backup.json”). Each file contains saved projects, automatic gear rules, device edits, preferences, favorites and runtime logs so you can copy them to external storage for offline safekeeping.
                    • Backup history ledger and the Latest activity board live inside Settings → Data & Storage . Use the Quick safeguards buttons there to trigger a full backup without switching tabs, then export the ledger alongside your archives so audit trails show the same timestamps even when you rotate media offline.
                    • Safety reminders sit beneath the Latest activity timeline. They flag when manual saves, automatic snapshots or full-app archives grow stale and include the captured timestamps so you know exactly what to refresh before relying on offline copies.
                    • Restore safeguards always create a brand-new safety backup before applying the file you choose, giving you an immediate rollback point if you need to undo the Restore process.
                    • Version history audits keep your change log organised. Toggle Compare versions to open the diff panel, pick two saves, review the highlighted summary, capture context in Incident notes and Export log for your archive.
                    • Restore rehearsals verify backups without touching production data. Launch Restore rehearsal , pick either the Full app backup or Project bundle mode, then click Choose file and watch the status banner as the sandbox loads. Review the comparison table and any automatic gear rule changes before staging the sandbox with Resume restore rehearsal . If anything looks unexpected, press Abort rehearsal to discard the sandbox safely; live data only changes after you finish with Restore .
                    • Combine the automatic layers with manual Export project and Device Database Export files when sharing work or moving between machines without internet access.

                    Project data safety checklist

                    1. Save the active setup under a new name before experimenting so earlier versions stay available for comparison.
                    2. Download both a full-app Backup and the current project JSON (via Export project ) immediately after important updates so you have self-contained copies you can move offline. Annotate filenames with the project, operator, location and timestamp to preserve context.
                    3. Store at least two copies of every file (for example, on your laptop and an external SSD) to guard against device loss when you are travelling or on set. Note where the redundant copies live so handoffs remain traceable.
                    4. Open each project JSON through Import project inside a private or secondary profile that stays offline to confirm it loads correctly; close the dialog or skip saving if you are only testing the archive.
                    5. Periodically test full-app backups by loading them with Restore rehearsal first. Check the comparison table and automatic rule preview, stage clean rehearsals with Resume restore rehearsal and cancel suspicious ones with Abort rehearsal . Only run Restore once the sandbox checks out—the planner still creates a fresh safety snapshot beforehand so you can revert instantly if needed. Log which backup you restored and where the validation copy lives.
                    6. Capture the runtime guard during every rehearsal. Open the developer console, confirm window.__cineRuntimeIntegrity.ok is true (or rerun window.cineRuntime.verifyCriticalFlows({ warnOnFailure: true })) and store the output with your verification notes so the audit trail proves every save/share/restore gateway stayed online-free.

                    Tip: Rename archived files with the project name and date (for example, “2024-06-15 camera-rig backup.json”) and mark trusted copies as read-only so the newest verified export stays obvious when you need to recover or share offline.

                    Console verification commands

                    • Open the developer console and inspect window.__cineRuntimeIntegrity. The guard should report { ok: true } with an empty missing list before you rely on local saves, shares or restores.
                    • Run window.cineRuntime.verifyCriticalFlows({ warnOnFailure: true }) whenever you finish a rehearsal or incident review. The refreshed report confirms every persistence, offline and UI safeguard stayed available without connectivity.
                    • Run window.cinePersistence.__internal.inspectAllBindings() to confirm every autosave, backup, share and restore wrapper resolves to a concrete implementation. Copy the frozen report into your verification notes so unresolved bindings surface before they can threaten stored projects.
                    • Use window.cinePersistence.storage.exportAllData() to preview the exact JSON captured by planner backups. Compare project counts, timestamps and file size before archiving exports to redundant media.
                    • Validate incoming bundles with window.cinePersistence.share.decodeSharedSetup(payload) before you import them on production data. Pair it with window.cinePersistence.share.applySharedSetup() inside a rehearsal profile to rehearse restores without touching live saves.

                    Tip: Capture the console output alongside your backup filenames so audits prove which commands you checked before travelling or rotating media.

                    Calculation Details

                    • Each device contributes its power draw in watts to Total Consumption, which you can inspect in the Power Summary panel.
                    • Current at 14.4 V (33.6 V for B‑Mount) and 12 V (21.6 V for B‑Mount) equals total watts divided by voltage.
                    • Runtime = battery capacity (Wh) ÷ total consumption.
                    • User-submitted runtimes collected through Submit User Runtime Feedback are adjusted for temperature and weighted before averaging:
                      • Temperature: ×2 at ≤−20 °C, ×1.6 at ≤−10 °C, ×1.25 at ≤0 °C.
                      • Resolution: ≥12K ×3, ≥8K ×2, ≥4K ×1.5, ≥1080p ×1; lower scales with 1080p.
                      • Frame rate scales linearly from 24 fps (e.g. 48 fps = ×2).
                      • Wi‑Fi adds 10 %; monitor entries below spec are weighted by their brightness ratio.
                      • Codec factors: RAW/BRAW/ARRIRAW/R3D/CinemaDNG/Canon RAW/X‑OCN ×1; ProRes ×1.1; DNx/AVID ×1.2; All‑Intra ×1.3; H.264/AVC ×1.5; H.265/HEVC ×1.7.
                      • The final weight reflects each device's share of total power draw.
                    • Warnings highlight when current approaches or exceeds main or D‑Tap output ratings—check the main output warning and D-Tap warning rows for specific advice.

                    How do I save a project?

                    Cine Power Planner autosaves every change to protected local storage. The status dot beside the Project Name field glows while that happens so you always know the latest update was captured. For a named snapshot:

                    1. Enter a descriptive name and click Save or press Enter / Ctrl+S (⌘+S on macOS).
                    2. Confirm it appears in the Saved Projects list. Loading another entry brings it back instantly while leaving the autosaved draft untouched.
                    3. Repeat the save action any time to update the snapshot, or choose -- New Project -- when you want to start fresh without deleting earlier saves.

                    Saved entries stay offline on your device, sync into manual exports and can always be restored, so no project data is lost between sessions.

                    How do I share or back up a project?

                    Use the exporter to create portable archives that protect every configuration detail:

                    1. Click Export Project and choose a secure file name. Include automatic gear rules when collaborators should inherit your automation presets.
                    2. Store the downloaded JSON bundle wherever you keep backups or send it to teammates. It contains the project requirements, gear list, runtime analysis, linked auto-backups and any custom devices needed to rebuild the setup.
                    3. To restore, paste or select the file inside the Import Project field and press Import. The planner validates the bundle and asks how to apply gear rules so nothing overwrites your library without approval.

                    Keep regular exports as part of your backup rotation; imports never delete existing saves, so you can maintain as many restore points as you need.

                    Layer the built-in safeguards so every archive stays verifiable offline:

                    • Visit Settings → Data & Storage and trigger the Quick safeguards buttons to capture a fresh full-app backup while the Latest activity timeline and backup history ledger log the run.
                    • Open Settings → Backup & Restore to watch the autosave status overlay, then use Compare versions to diff manual saves against the latest auto-backup and Export log entries for your paper trail.
                    • Rehearse every backup by loading it with Restore rehearsal before you apply Restore . The sandbox confirms every project, gear rule and favorite without touching live data.

                    Every export is human-readable JSON—store redundant copies on offline media, note their filenames in your verification log and archive the diff summaries so crews travelling without connectivity inherit the same history.

                    How do automatic backups work?

                    Cine Power Planner layers three automatic safety nets alongside your manual saves:

                    • Auto snapshots capture the active project every 10 minutes or after roughly 50 edits, and rapid adjustments are batched so bursts closer than a second only advance the counter once. When you switch projects, import or export data or prepare to reload, the planner captures an immediate snapshot even if that cadence hasn't elapsed. Enable Show auto backups in project list in Settings → Backup & Restore to reveal the timestamped auto-backup-… entries beneath the Saved Projects dropdown so you can load, review and rename them.
                    • Hourly full-app backups download JSON archives (for example, “2024-06-15T14-00-00 full app backup.json”) that bundle every project, automatic gear rule, device edit, favorite and runtime log. Keep at least two copies on separate drives so you always have an offline recovery point.
                    • Restore safeguards run whenever you restore or perform a factory reset. The planner creates a fresh “before” backup automatically so you can revert immediately if the new file is not what you expected.

                    Open any auto snapshot to inspect it; press Save if you want to keep the snapshot as a regular project. Archive the hourly downloads once verified, or delete them after you have stored redundant copies elsewhere.

                    Keep Settings → Backup & Restore open during rehearsals to monitor the autosave status overlay, launch Compare versions and record Export log results alongside your backups. Combine that diff history with Restore rehearsal runs and the Latest activity reminders in Settings → Data & Storage so every safeguard proves it captured a fresh copy before you head offline.

                    How is battery runtime calculated?

                    The app divides the selected battery's capacity (in watt-hours) by the total power consumption of your devices. For example, a 98 Wh battery powering a 24 W project is estimated to run for about 4 hours.
